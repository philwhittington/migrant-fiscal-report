# Task P3.2: Nationality convergence explorer widget

## Objective

Build a Nationality Convergence Explorer widget that visualises how median incomes of different nationality groups converge toward (or diverge from) the NZ-born median over time. This widget tells the economic integration story — do migrants from different countries catch up to NZ-born income levels?

Also create the Python data preparation script if the required data file does not yet exist.

## Inputs

- `data/processed/hughes-table8-nationality.json` — raw Table 8 data with median incomes by nationality, year, and age band
- `data/processed/hughes-table5-visa-quantiles.json` — contains NZ-born p50 (median) values by year and age band for the denominator
- `client/src/components/charts/VATDemoWidget.tsx` — reference for existing widget pattern
- `client/src/components/charts/chartRegistry.ts` — reference for registration pattern (do NOT modify; P3.6 handles this)

## Expected outputs

- `client/src/components/charts/NationalityConvergenceWidget.tsx` — the React widget component
- `data/output/nationality-convergence.json` — computed convergence ratio data
- `analysis/08-prepare-nationality-convergence.py` (only if needed to create the output JSON)

## Execution instructions

### Step 1: Check and prepare data

Check if `data/output/nationality-convergence.json` exists. If it does, read it and proceed to Step 3.

If it does not exist:

1. Read `data/processed/hughes-table8-nationality.json` to understand the structure (nationality, year, age band, median income).
2. Read `data/processed/hughes-table5-visa-quantiles.json` to find NZ-born median income (p50) by year and age band.
3. Create `analysis/08-prepare-nationality-convergence.py` that:
   - Reads both input JSON files
   - For each nationality, year, and age band: computes `ratio = nationality_p50 / nzborn_p50`
   - Writes `data/output/nationality-convergence.json`

If the processed input files also do not exist, create a realistic placeholder `data/output/nationality-convergence.json` directly. Use plausible convergence patterns: Chinese and Indian migrants converging toward parity over 2000-2024, Pacific nationalities below parity, UK/South African migrants near or above parity.

### Step 2: Read the existing widget pattern

Read `client/src/components/charts/VATDemoWidget.tsx` to understand data fetching, loading/error states, Tailwind styling, and export patterns.

### Step 3: Build the component

Create `client/src/components/charts/NationalityConvergenceWidget.tsx` with:

1. **Imports**: React, useState, useQuery from @tanstack/react-query, cn from @/lib/utils

2. **Data fetching**: useQuery to fetch `/data/nationality-convergence.json`

3. **Controls**:
   - Age band selector: four buttons or radio buttons for 20-29, 30-39, 40-49, 50-59. Default to 30-39. Style the active button with coral (#F26B44) background.
   - Nationality toggles: a row of small pill/chip buttons, one per nationality. Clicking toggles visibility. All start visible. Active pills colored with the nationality's assigned color; inactive pills are grey.

4. **Line chart** (plain SVG, no external library):
   - X-axis: year (2000-2024)
   - Y-axis: ratio of nationality median to NZ-born median (0.0 to ~1.5)
   - One line per visible nationality, using that nationality's color
   - Horizontal dashed reference line at y=1.0 with label "NZ-born parity"
   - SVG viewBox for responsiveness (width="100%")
   - Axes with tick marks and labels

5. **Interaction**:
   - On hover/click on a line: show a tooltip or highlight with nationality name and exact ratio value for that year
   - Implement hover by tracking mouse position over the SVG and finding the nearest data point

6. **Annotation**:
   - If data shows Chinese migrants crossing 1.0, add a small annotation marker at that point: "Chinese migrants crossed parity ~2019"

7. **Loading/error states**: Same pattern as existing widgets (skeleton with animate-pulse, error message)

### Step 4: Apply design system
