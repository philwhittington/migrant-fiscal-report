# Task P5.1: Data QA — verify numbers against source

## Objective

Systematically verify every numerical claim in the report markdown against the source JSON data files. This ensures no numbers were hallucinated, rounded incorrectly, or taken out of context. The output is a QA report documenting every check.

## Inputs

- `content/migrant-fiscal-impact.md` — the report markdown with numerical claims
- `data/output/*.json` — widget data files
- `data/processed/*.json` — processed source data
- `data/raw/` — raw source data (if available, for cross-checking)

## Expected outputs

- `.agent/qa-data-report.md` — a structured QA report documenting every numerical claim and its verification status

## Execution instructions

### Step 1: Read the report

Read `content/migrant-fiscal-impact.md` in full.

### Step 2: Extract all numerical claims

Go through the report line by line and extract every instance of:
- Percentages (e.g., "85% of skilled migrants")
- Dollar amounts (e.g., "$250,000 NPV", "$15,400 net contribution")
- Counts (e.g., "100,000 people", "45,000 initial cohort")
- Ratios (e.g., "1.15 times the NZ-born median")
- Year references tied to data (e.g., "by 2019, Chinese migrants crossed parity")
- Ranges (e.g., "between $180,000 and $320,000")

For each claim, record:
- The exact text from the report
- The section/heading it appears under
- The line number (approximate)

### Step 3: Trace each claim to source data

For each numerical claim:

1. Identify which data file should contain the source value. The mapping is:
   - NPV values -> `data/output/npv-by-visa-age.json`
   - Fiscal components -> `data/output/fiscal-components-by-migrant-type.json`
   - Nationality convergence ratios -> `data/output/nationality-convergence.json`
   - RV2021 composition numbers -> `data/output/rv2021-composition.json`
   - Retention rates -> `data/processed/retention-curves-by-visa.json`
   - Raw table data -> `data/processed/hughes-table*.json`

2. Read the relevant data file.

3. Locate the specific value. Check if the claim matches the data:
   - Exact match: PASS
   - Within rounding tolerance (e.g., claim says "85%" and data is 84.7%): PASS with note
   - Does not match: FAIL -- record the discrepancy
   - Cannot find source: UNVERIFIED -- the data file may not contain this value

### Step 4: Check for suppressed data issues

For any numbers based on small cell counts (e.g., fewer than 6 individuals in a nationality-age-year cell), flag them. Small cell counts mean:
- The number may be unreliable
- It may need suppression for privacy if based on individual-level data

### Step 5: Check for internal consistency

Verify that:
- Percentages within a group sum to ~100% (within rounding)
- Counts match totals (e.g., if the report says "700,000 total migrants" and lists subcategories, the subcategories should sum to 700,000)
- NPV values are consistent between the executive summary and the body

Create `.agent/qa-data-report.md` (create the `.agent/` directory if it does not exist):

## Self-check

1. Every percentage, dollar amount, count, and ratio in the report has been checked.
2. The QA report is complete with all sections filled.
3. Any FAIL items are clearly documented with the expected vs actual values.
4. The `.agent/` directory exists and the report is readable.
5. No claims are silently skipped.
