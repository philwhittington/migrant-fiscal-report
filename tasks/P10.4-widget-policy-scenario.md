# Task P10.4: Policy scenario explorer widget

## Objective

Build an interactive policy scenario widget with three sliders that let users explore how changes to migration policy settings affect aggregate fiscal outcomes. This is the most analytically ambitious widget — it performs real-time reweighting of pre-loaded per-visa-type distributions in the browser, enabling "what if" analysis without server-side computation. It answers questions like "what happens to the fiscal bottom line if we increase the skilled visa share from 40% to 60%?"

## Inputs

- `data/output/synth-npv-distributions.json` — per-visa-type NPV distributions (full histogram data for reweighting)
- `data/output/synth-population-summary.json` — baseline population composition and aggregate statistics
- `client/src/components/charts/NPVCalculatorWidget.tsx` — reference for widget pattern
- `client/src/components/charts/chartRegistry.ts` — reference for lazy import pattern (do NOT modify; P10.5 handles registration)

## Expected outputs

- `client/src/components/charts/PolicyScenarioWidget.tsx` — a single self-contained React component

### Step 3: Design the computation logic

The widget performs real-time reweighting in the browser. The logic:

1. **Skilled visa share slider** (10-80%, default matches baseline): changes the weight of the Skilled distribution relative to others. Other categories scale proportionally to sum to 100%.

2. **Retention change slider** (+/-10 percentage points at year 10): shifts all retention curves up or down. This affects the per-individual NPV by changing how many years of fiscal contribution are counted. Apply as a linear scaling factor to each individual NPV: `adjusted_npv = npv * (1 + retention_delta / baseline_retention)`. This is an approximation — document it.

3. **NZ Super threshold slider** (5/10/15/20 years): changes the eligibility threshold. Pre-compute NPV adjustments for each threshold by removing/adding NZ Super years from the NPV. Use the per-year NZ Super cost from `synth-population-summary.json` if available, else use a constant ($20,000/year as approximate NZ Super annual cost).

For each slider position, recompute:
- Aggregate fiscal impact per 1,000 migrants
- Percentage who are net contributors
- Total NPV (extrapolated from per-1,000 figure)

### Step 4: Build the component

Create `client/src/components/charts/PolicyScenarioWidget.tsx` with:

1. **Imports**: React, useState, useMemo, useQuery from @tanstack/react-query, cn from @/lib/utils

2. **Data fetching**: Two useQuery calls for both JSON files

3. **Three sliders**:
   - **Skilled visa share**: range input, 10-80%, step 5%, default from baseline. Label shows current value. Styled with Tailwind (accent color teal).
   - **Retention change**: range input, -10 to +10 percentage points, step 1pp, default 0. Label shows "+X pp" or "-X pp".
   - **NZ Super threshold**: range input or segmented selector with values 5/10/15/20 years, default 10.

4. **Output panel** — three large metric cards in a row:
   - Card 1: "Aggregate fiscal impact per 1,000 migrants" — show dollar value and delta from baseline
   - Card 2: "Net contributors" — show percentage and delta
   - Card 3: "Total NPV" — show dollar value and delta

5. **Delta display**:
   - For each metric, show baseline value in steel text
   - Show current value in large navy text
   - Show delta as a chip: green (sage) with upward arrow if positive, coral with downward arrow if negative
   - Delta formatted as "+$X.Xm" or "-X.X pp"

6. **Baseline indicator**: When all sliders are at default, show "Current settings" label. When any slider is moved, show "Modified scenario" with a "Reset to baseline" button.

7. **Computation**: Use useMemo to recompute outputs when any slider changes. The computation should be fast enough for real-time interaction (no debouncing needed for simple arithmetic).

8. **Loading state**: Skeleton
9. **Error state**: Styled error message
