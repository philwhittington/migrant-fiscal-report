# Task P3.1: NPV lifecycle calculator widget

## Objective

Build an interactive NPV Lifecycle Calculator widget that lets users explore the net present value of fiscal contributions by visa type and arrival age. This is the centrepiece interactive element of the report, showing whether different migrant types are net contributors or net costs over their lifetime in New Zealand.

## Inputs

- `data/output/npv-by-visa-age.json` — NPV values indexed by visa type and arrival age, with year-by-year cumulative fiscal contribution series
- `data/output/fiscal-components-by-migrant-type.json` — decomposition of fiscal assumptions (retention rates, tax premiums, NZ Super eligibility) by migrant type
- `client/src/components/charts/VATDemoWidget.tsx` — reference for the existing widget pattern (imports, data fetching, Tailwind styling, loading/error states)
- `client/src/components/charts/chartRegistry.ts` — reference for how widgets are registered (do NOT modify this file; P3.6 handles registration)

## Expected outputs

- `client/src/components/charts/NPVCalculatorWidget.tsx` — a single self-contained React component

### Step 2: Read the data files

Read both JSON data files. Understand the structure:
- `npv-by-visa-age.json` should contain an array or object keyed by visa type, each with arrival age entries. Each entry should have a `cumulativeSeries` array (year-by-year cumulative NPV) and a `totalNPV` number.
- `fiscal-components-by-migrant-type.json` should contain retention rates, tax premium percentages, and NZ Super eligibility flags per migrant type.

If either file does not exist yet, create a realistic placeholder JSON file with the correct structure so the widget can be developed and tested. Use plausible values based on NZ fiscal research (skilled migrants are typically net positive, humanitarian migrants net negative in early years). Document what you assumed in a comment at the top of the component file.

### Step 3: Build the component

Create `client/src/components/charts/NPVCalculatorWidget.tsx` with:

1. **Imports**: React, useState, useQuery from @tanstack/react-query, cn from @/lib/utils

2. **Data fetching**: Two useQuery calls, one for each JSON file. URLs: `/data/npv-by-visa-age.json` and `/data/fiscal-components-by-migrant-type.json`

3. **Controls**:
   - Arrival age slider: range 20-55, step 5, styled with Tailwind. Show current value.
   - Visa type dropdown: `<select>` with options for Skilled, Family, Humanitarian, Australian, Temp Work, Student, NZ-born baseline.

4. **Main display**:
   - Large NPV value text. Use `text-4xl font-bold`. Color it green (#7A9E7E / sage) if net positive, coral (#F26B44) if net negative.
   - Label: "Net present value of lifetime fiscal contribution"

5. **Line chart** (plain SVG, no external library):
   - X-axis: years since arrival (0 to length of cumulative series)
   - Y-axis: cumulative NPV in NZD
   - Primary line: selected visa type + arrival age, colored with teal (#6EA1BE)
   - Secondary line: NZ-born baseline at same age, colored with steel (#6F899A), dashed
   - Axes with tick labels
   - Legend showing both lines
   - SVG should be responsive (use viewBox, width="100%")

6. **Decomposition table** below the chart:
   - Rows: Retention rate at yr+5, yr+10, yr+15 | Tax premium vs NZ-born | NZ Super eligibility | Key fiscal components
   - Styled as a simple Tailwind table with alternating row shading

7. **Loading state**: Skeleton placeholder with animate-pulse
8. **Error state**: Styled error message

### Step 4: Apply design system colors

- Background: bg-[#F8F3EA] or equivalent off-white
