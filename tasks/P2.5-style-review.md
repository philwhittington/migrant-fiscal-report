# Task P2.5: Apply HW report style review to the full draft

## Objective

Perform a comprehensive style and quality review of the complete migrant fiscal impact report. This is the final content gate before integration with widgets. The output must meet Heuser|Whittington publication standards: precise, evidence-based, professionally hedged, with no advocacy. Every numerical claim must trace to a source. Every heading must be sentence case. The prose must be clear enough for a senior Treasury official to read without confusion.

## Execution instructions

### Step 1: Read the full report

1. Read `.agent/SCRATCHPAD.md` for current state.
2. Read `content/migrant-fiscal-impact.md` in full — every section, including the executive summary and references.
3. Read the key data files so you can verify numerical claims in the text.

### Step 2: Heading audit

Scan every heading in the document. Apply these rules:

- **Sentence case only**: Capitalise the first word and proper nouns only. Examples:
  - Correct: "The aggregate picture"
  - Correct: "Nationality and economic integration"
  - Correct: "The 2021 resident visa effect"
  - Wrong: "The Aggregate Picture"
  - Wrong: "Fiscal Selection by Visa Type"
- **Numbered sections**: Keep the number prefix (e.g., "## 3. The aggregate picture")
- **Subsection headings**: Same rule applies (e.g., "### 2.1 Data sources" not "### 2.1 Data Sources")

Fix any heading that does not comply.

### Step 3: Precision audit

Read every sentence that contains a number. For each:

1. **Verify the number against source data.** Read the relevant JSON file and confirm the number matches. If there is a discrepancy, update the text to match the data (data is the source of truth).
2. **Verify source attribution.** Every number must name its source. Acceptable forms:
   - "Hughes AN 26/02 shows that..."
   - "(Hughes, Table 1)"
   - "According to our NPV model..."
   - "Wright and Nguyen (2024) estimate..."
   Unacceptable: a bare number with no source mentioned in the same paragraph.
3. **Verify hedging.** Estimates and model outputs should use hedging language:
   - "approximately $189,000"
   - "our estimates suggest"
   - "the data indicates"
   Not: "$189,000" stated as fact, or "migrants contribute $189,000"

Fix any sentence that fails these checks.

### Step 4: Language quality audit

Read every paragraph and check against these rules:

**Vague qualifiers — eliminate:**
Search for and replace these words with precise alternatives:
- "significant" → state the magnitude or use "substantial" only if quantified
- "considerable" → state the amount
- "major" → specify what makes it major
- "very" → remove or quantify
- "clearly" → remove (if it were clear, you would not need to say so)
- "obviously" → remove
- "interesting" / "interestingly" → remove (let the reader decide)
- "important" → say why, or remove

**Advocacy language — remove:**
Search for and remove or rephrase:
- "should" (when prescribing policy)
- "must" (when prescribing action)
- "need to" (when prescribing)
- "it is essential that" → rephrase as observation
- "policy-makers should consider" → "this finding is relevant to..."
- "we recommend" → "a natural extension would be..."

Exception: "should" is acceptable when describing methodological choices ("future work should use IDI data") or when hedging ("this should be interpreted with caution").

**Active voice — prefer:**
Convert passive voice to active where it improves clarity:
- "It was found that migrants pay..." → "We find that migrants pay..." or "The data shows that migrants pay..."
- "The analysis was conducted using..." → "We conducted the analysis using..."
- Exception: passive is acceptable when the agent is unimportant ("Tax data was obtained from Inland Revenue")

**Paragraph length — enforce:**
No paragraph should exceed 6 sentences. If a paragraph is longer, split it at a natural break point. Most paragraphs should be 3-5 sentences.

### Step 5: Structural coherence check

Read the report as a whole and check:

1. **Section transitions**: Each section should end with 1-2 sentences that motivate the next section. Check that these transitions exist and flow logically. The narrative arc should be:
   - Introduction → why this matters
   - Methodology → how we do it
   - Aggregate picture → the headline (migrants over-contribute)
   - Visa type → but it varies enormously by pathway
   - RV2021 → a recent policy shock changed the composition
   - Nationality → convergence over time is real and measurable
   - NPV → putting it all together into lifecycle estimates
   - Caveats → honest about limitations
   - Conclusion → what comes next

2. **Executive summary accuracy**: Every number in the executive summary must appear in the body. Every finding mentioned must be supported by a section. If the executive summary claims something that the body does not support, either add support in the body or remove the claim from the summary.

3. **Internal consistency**: Check that numbers cited in one section are consistent with the same numbers cited in other sections. For example, if Section 3 says "38% of income tax" and the Executive summary says "37%", one of them is wrong. The data file is the tiebreaker.

4. **Widget placement**: Verify all 5 widget divs are still present and correctly placed:
   - `<div data-demo="fiscal-waterfall"></div>` in Section 3
   - `<div data-demo="retention-explorer"></div>` in Section 4
   - `<div data-demo="rv2021-shift"></div>` in Section 5
   - `<div data-demo="nationality-convergence"></div>` in Section 6
   - `<div data-demo="npv-calculator"></div>` in Section 7

### Step 6: South Asia framing check

Read the paragraph(s) about South Asian migrants specifically. This is a reputational risk area — the text must:
- State facts neutrally
- Provide context (visa pathway composition, group size)
- Not use deficit framing ("only pay X" → "pay X, reflecting the high proportion of family-stream entrants in this group")
- Not compare unfavourably to other groups without context
- Acknowledge non-tax contributions where appropriate

If the framing is not neutral enough, rewrite the relevant paragraph(s).

### Step 7: References check

Verify the References section includes at minimum:
- Hughes, T. (2026). *Personal income tax paid in New Zealand by migrants.* New Zealand Treasury Analytical Note AN 26/02.
- Wright, M. and Nguyen, D. (2024). *The fiscal incidence of New Zealand's tax and spending: 2018/19 update.* New Zealand Treasury Analytical Note AN 24/09.
- Varela, P., Husek, N., McDonald, S., and Converse, B. (2021). *Estimating the fiscal impact of immigration: An analysis framework.* Australian Treasury Working Paper 2021-01.

If any other sources are cited in the body, they must appear in References. Add any that are missing.

### Step 8: Final read-through

Read the entire report one final time, from title to references. Focus on:
- Does it read as a single coherent document, not a patchwork of separately written sections?
- Is the tone consistent throughout (rigorous, evidence-based, professionally hedged)?
- Would you be comfortable sending this to a Treasury Principal Advisor?

Make any final adjustments.
