# Task P5.3: Widget QA — test rendering and build

## Objective

Verify that all widgets compile, the build succeeds, data files are correctly wired, and the chart registry is complete. This is the integration test for the widget layer.

## Inputs

- `client/src/components/charts/NPVCalculatorWidget.tsx`
- `client/src/components/charts/NationalityConvergenceWidget.tsx`
- `client/src/components/charts/RetentionExplorerWidget.tsx`
- `client/src/components/charts/RV2021ShiftWidget.tsx`

- `client/src/components/charts/chartRegistry.ts`
- `script/generate-static-data.ts`
- `data/output/*.json`
- `data/processed/*.json`
- `package.json`

## Expected outputs

- `.agent/qa-widget-report.md` — QA report documenting all checks
- Any bug fixes made to widget files during QA

## Execution instructions

### Step 1: Install dependencies

```bash
cd /Users/PhilWhittington/Projects/migrant-fiscal-report
npm install
```

If `npm install` fails, read `package.json` to understand the dependency requirements and fix any issues.

### Step 2: Generate static data

```bash
cd /Users/PhilWhittington/Projects/migrant-fiscal-report
npx tsx script/generate-static-data.ts
```

Check the output for warnings. Verify that `client/public/data/` contains:
- `report.json`
- `npv-by-visa-age.json`
- `fiscal-components-by-migrant-type.json`
- `nationality-convergence.json`
- `rv2021-composition.json`
- `retention-curves-by-visa.json`

List all files:
```bash
ls -la client/public/data/
```

For any missing files, note them. The widget will show a loading error at runtime but should not crash.

### Step 3: TypeScript compilation check

```bash
cd /Users/PhilWhittington/Projects/migrant-fiscal-report
npx tsc --noEmit
```

If there are errors:
1. Read each error message carefully.
2. Open the file and line referenced.
3. Fix the error. Common issues:
   - Missing type annotations
   - Incorrect import paths
   - Missing exports (the registry expects named exports like `NPVCalculatorWidget`)
   - Using undefined variables
   - Type mismatches in data structures
4. Re-run `npx tsc --noEmit` after each fix until clean.

### Step 4: Build check

```bash
cd /Users/PhilWhittington/Projects/migrant-fiscal-report
npm run build
```

If the build fails:
1. Read the error output.
2. Fix the issue (common: Vite build errors for unresolved imports, CSS issues, SVG syntax errors in JSX).
3. Re-run until successful.

### Step 5: Verify widget files

For each of the 5 widget files, check:

1. **File exists**:
   ```bash
   ls client/src/components/charts/NPVCalculatorWidget.tsx
   ls client/src/components/charts/NationalityConvergenceWidget.tsx
   ls client/src/components/charts/RetentionExplorerWidget.tsx
   ls client/src/components/charts/RV2021ShiftWidget.tsx
   ls client/src/components/charts/FiscalWaterfallWidget.tsx
   ```

2. **Has named export**: Search each file for the export statement:
   ```bash
   grep "export function" client/src/components/charts/NPVCalculatorWidget.tsx
   ```
   Should return `export function NPVCalculatorWidget()`.

3. **Has React import and useQuery**: Each widget should import React and useQuery.

4. **Has loading state**: Search for "loading" or "animate-pulse" or similar loading indicator.

5. **Has error state**: Search for "error" or "Error" to verify error handling exists.

### Step 6: Verify chartRegistry

Read `client/src/components/charts/chartRegistry.ts` and verify:
- It contains exactly 5 entries (no old Pacific LLM entries)
- The demo IDs are: npv-calculator, nationality-convergence, retention-explorer, rv2021-shift, fiscal-waterfall
- Each import path matches the actual file name
- Each named export matches the function name in the widget file

### Step 7: Verify data files exist in public

After running generate-static-data.ts, check:
```bash
ls client/public/data/*.json
```

Cross-reference: each widget fetches a specific URL. Verify the mapping:
- NPVCalculatorWidget -> `/data/npv-by-visa-age.json` AND `/data/fiscal-components-by-migrant-type.json`
- NationalityConvergenceWidget -> `/data/nationality-convergence.json`
- RetentionExplorerWidget -> `/data/retention-curves-by-visa.json`
- RV2021ShiftWidget -> `/data/rv2021-composition.json`
- FiscalWaterfallWidget -> `/data/fiscal-components-by-migrant-type.json`
