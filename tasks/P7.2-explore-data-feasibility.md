# Task P7.2: Explore data feasibility and Census API decision

## Objective
Systematically assess all processed data files to determine: (a) what attributes can be reliably assigned to synthetic individuals, (b) the sparsity and suppression rates in each table, and (c) whether pulling Census cross-tabs from the Stats NZ API would materially improve the synthetic population. Document the decision and reasoning.

## Inputs
- All files in `data/processed/*.json`
- `.agent/SCRATCHPAD.md`

## Expected outputs
- Updated `.agent/SCRATCHPAD.md` with "## Census API decision" section filled in
- A brief data feasibility summary in SCRATCHPAD

## Execution instructions

### Step 1: Read SCRATCHPAD
Read `.agent/SCRATCHPAD.md` for context.

### Step 2: Assess each processed data file

For each JSON file in `data/processed/`, load it and compute:
1. Total number of records
2. Number of unique cells (unique combinations of dimension variables)
3. Minimum cell count (smallest non-suppressed count)
4. Suppression rate (% of cells where original data was suppressed, if detectable)
5. Dimension variables and their cardinalities

Print a summary table:
```
File                          Records  Cells  Min_N  Dimensions
hughes-table4-visa-sub...     5,145    X      Y      year(25) × age(10) × visa_sub(34)
hughes-table5-visa-quant...   11,100   X      Y      year(25) × age(10) × visa(10) × quantile(5)
...
```

### Step 3: Assess income characterisation quality

For the income fitting task (P8.2), we need Table 5 quantiles (p10-p90) per visa_category × age_start cell. Check:
- How many unique (visa_category, age_start) cells exist for tax year 2019?
- Do all 5 quantile values exist for each cell?
- Are any quantile values suspiciously small (potential suppression)?
- Can we fit a distribution to all cells, or are some too sparse?

### Step 4: Census API decision

The question: would adding occupation/industry cross-tabs from Stats NZ Census 2023 API materially improve the fiscal estimates?

Consider:
- **Pro**: occupation and industry add realism to the synthetic population. They could improve wage distributions (e.g., nurses vs IT workers within "skilled" visa category).
- **Con**: Hughes's quantile data (p10-p90) already captures the within-group income distribution. Adding occupation doesn't change the fiscal calculation — tax is computed from income, not occupation. It adds colour but not fiscal precision.
- **Con**: Census API requires an API key, adds a dependency, and the data is for the total population (not migrants specifically). Cross-tabulating Census occupation data with Hughes visa data requires assumptions about the occupation distribution of migrants.
- **Recommendation**: "no" unless the Table 5 quantile data is substantially sparser than expected.

Write the decision to SCRATCHPAD.md:
```markdown
## Census API decision
- Decision: no
- Reasoning: [your analysis]
- Quantile coverage: [X] cells with full p10-p90 data for 2019
- Income characterisation quality: [assessment]
```

### Step 5: Summarise feasibility

Write to SCRATCHPAD.md:
```markdown
## Data feasibility summary
- Seed population (Table 4, 2019): [N] visa_sub × age cells, total count [X]
- Income distributions (Table 5, 2019): [N] visa × age cells with full quantiles
- Nationality assignment (Table 10): [N] nationality × relationship cells
- Tenure distribution (Table 11): [N] relationship × age × tenure cells
- Retention curves: [N] visa types with [X] years of data
- Overall: [feasible/marginal/problematic]
```
