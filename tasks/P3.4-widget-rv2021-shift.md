# Task P3.4: RV2021 composition shift widget

## Objective

Build a widget that visualises the Residence Visa 2021 (RV2021) one-off amnesty and its effect on the migrant stock composition. RV2021 moved approximately 100,000 people from temporary to resident visa status in New Zealand. This widget shows the before/after composition shift with side-by-side stacked bar charts.

Also create the Python data preparation script if the required data file does not yet exist.

## Inputs

- `data/output/rv2021-composition.json` — before/after composition by visa category
- `data/processed/hughes-table4-*.json` — raw Table 4 data (visa category counts by year) if the output file needs to be computed
- `client/src/components/charts/VATDemoWidget.tsx` — reference for existing widget pattern
- `client/src/components/charts/chartRegistry.ts` — reference for registration (do NOT modify; P3.6 handles this)

## Expected outputs

- `client/src/components/charts/RV2021ShiftWidget.tsx` — the React widget component
- `data/output/rv2021-composition.json` — composition data
- `analysis/09-prepare-rv2021.py` (only if needed to create the output JSON)

## Execution instructions

### Step 1: Check and prepare data

Check if `data/output/rv2021-composition.json` exists. If it does, read it and proceed to Step 3.

If it does not exist:

1. Check for any Table 4 data files in `data/processed/` (glob for `hughes-table4*`).
2. If Table 4 data exists, create `analysis/09-prepare-rv2021.py` that:
   - Reads the Table 4 data
   - Extracts the composition snapshot at 2021 (before RV2021) and 2024 (after)
   - Computes counts and percentages by visa subcategory
   - Writes `data/output/rv2021-composition.json`
3. If no Table 4 data exists, create a realistic placeholder `data/output/rv2021-composition.json` directly.

### Step 2: Read the existing widget pattern

Read `client/src/components/charts/VATDemoWidget.tsx` for data fetching, loading/error states, Tailwind styling, and export patterns.

### Step 3: Build the component

Create `client/src/components/charts/RV2021ShiftWidget.tsx` with:

1. **Imports**: React, useState, useMemo, useQuery from @tanstack/react-query, cn from @/lib/utils

2. **Data fetching**: useQuery to fetch `/data/rv2021-composition.json`

3. **Controls**:
   - Age band filter: dropdown or button group for "All ages", "20-29", "30-39", "40-49", "50-59". Default: "All ages".

4. **Stacked bar charts** (plain SVG, no external library):
   - Two vertical stacked bars side by side, labelled "2021" and "2024"
   - Each bar segmented by visa category using the category colors
   - Bar height proportional to total count
   - Each segment shows: category name (if segment is large enough), count, and percentage
   - Bars should be wide enough to read labels (at least 120px each)
   - Small gap between the two bars

5. **Movement indicators**:
   - Between or below the bars, show annotations for the largest changes:
     - Arrows or lines connecting segments that grew (green arrow up) or shrank (coral arrow down)
     - Or a simple summary row: "Temp Work: 165k -> 85k (-48%)" / "Skilled: 185k -> 238k (+29%)"

6. **Key stat callout**:
   - Below the chart, a highlighted box: "Approximately 100,000 people moved from temporary to resident visa status under RV2021"
   - Styled with coral (#F26B44) left border, bg-[#F8F3EA]

7. **Tooltips/hover**:
   - Hovering a segment shows: category name, count, percentage of total

8. **Loading/error states**: Skeleton with animate-pulse, error message

### Step 4: Apply design system

- Container: bg-white or bg-[#F8F3EA] rounded-lg p-6
- Bar segment colors: as defined in the JSON data colors object
- Labels: fill="#152F43" (navy)
- Percentage text: fill="#6F899A" (steel), smaller font
- Key stat box: border-l-4 border-[#F26B44] bg-[#F8F3EA]/50 p-4
- Movement arrows: positive changes in #7A9E7E (sage), negative in #F26B44 (coral)

### Step 5: Export correctly
