# Task P9.2: Add distributional findings to report sections 3-7

## Objective

Enhance Sections 3-7 of the report content with distributional claims derived from the synthetic population outputs. This transforms the report from presenting only group means to also showing the spread and probability of individual fiscal outcomes, making the analysis substantially more informative for policy audiences.

## Inputs

- `content/migrant-fiscal-impact.md` — the main report file; Sections 3-7 contain findings by visa type, nationality, fiscal components, retention, and NPV
- `data/output/synth-npv-distributions.json` — per-visa-type NPV distributions with percentiles, probability of net contribution, inequality metrics
- `data/output/synth-fiscal-distributions.json` — distributional data for individual fiscal components (tax, health, education, benefits, NZ Super)
- `data/output/synth-household-npv.json` — family-unit-level NPV distributions
- `data/output/synth-population-summary.json` — aggregate summary statistics for the synthetic population

## Expected outputs

- `content/migrant-fiscal-impact.md` — updated with 1-2 new paragraphs in each of Sections 3, 4, 5, 6, and 7 (approximately 800 words total across all sections)

## Execution instructions

### Step 1: Read all synthetic output files

Read each JSON file and build a reference table of key statistics:

From `synth-npv-distributions.json`, extract for each visa type:
- p10, p25, median, p75, p90 of individual NPV
- Percentage who are net contributors (NPV > 0)
- Percentage who are net contributors exceeding $100k
- Gini coefficient or interquartile range of NPV outcomes
- Mean vs median comparison (skewness indicator)

From `synth-fiscal-distributions.json`, extract:
- p25-p75 range for each fiscal component by visa type
- Which components have the widest distributional spread

From `synth-household-npv.json`, extract:
- Family-unit NPV by configuration (primary alone, +spouse, +children)
- How family composition shifts the net contribution picture

From `synth-population-summary.json`, extract:
- Total synthetic population size
- Aggregate fiscal impact
- Share of net contributors across all visa types combined

### Step 2: Read existing Sections 3-7

Read each section to understand what claims already exist and where distributional additions fit naturally. Identify the paragraph after which new content should be inserted in each section.

### Step 3: Draft distributional paragraphs

For each section, add 1-2 paragraphs with specific distributional statistics. Follow these templates:

**Section 3 (Fiscal components):** Add distributional spread for key components. Example pattern: "While the average skilled migrant pays $X in income tax, the p10-p90 range spans $A to $B, reflecting substantial heterogeneity in earnings. The health expenditure distribution is notably tighter, with a p10-p90 range of $C to $D, consistent with the relatively uniform public health cost structure."

**Section 4 (Visa type analysis):** Add probability statements per visa type. Example pattern: "Our synthetic population estimates suggest that 72% of skilled migrants arriving at age 30 are net contributors exceeding $100k over their lifetime, with a p10-p90 range of $X to $Y. By contrast, only Z% of humanitarian migrants in the same age cohort exceed this threshold."

**Section 5 (Nationality):** Add distributional comparisons across nationality groups. Use median-vs-mean to highlight skewness.

**Section 6 (Retention effects):** Add how distributional outcomes change with retention assumptions. Probability statements about how retention uncertainty propagates to NPV uncertainty.

**Section 7 (NPV summary):** Add household-level findings. "A skilled family unit of four — primary applicant, spouse, and two children — still contributes a net $X, though the distribution is wider than for individuals alone."

### Step 4: Source every number

For every number inserted:
1. Confirm it exists in one of the four JSON files
2. Note the exact key path (e.g., `synth-npv-distributions.Skilled.arrivalAge30.p90`)
3. Do not round aggressively — use the precision from the source file
4. If a number does not exist in the data files, do not insert it

### Step 5: Apply appropriate hedging language

Every distributional claim must use language appropriate for synthetic estimates:
- Use: "our synthetic population estimates suggest", "the model indicates", "estimated distribution shows"
- Do not use: "migrants contribute", "the data shows", "we observe"
- Make clear these are model-generated, not observed individual outcomes

### Step 6: Review integration

Read each modified section in full to confirm:
- New paragraphs flow naturally from existing prose
- No contradiction between Phase 1 group means and Phase 2 distributions
- Distributional claims are additive (they enrich, not replace, the existing analysis)
