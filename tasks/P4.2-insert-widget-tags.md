# Task P4.2: Insert widget data-demo tags into report markdown

## Objective

Insert the `<div data-demo="widget-id"></div>` tags at the correct positions in the report markdown so that the MarkdownRenderer will lazy-load each widget when the reader scrolls to that section. Each widget must appear exactly once, in a location that supports the surrounding narrative.

## Inputs

- `content/migrant-fiscal-impact.md` — the report markdown (created in Phase 2)
- Knowledge of the 5 widget demo IDs:
  - `npv-calculator` — NPV Lifecycle Calculator
  - `nationality-convergence` — Nationality Convergence Explorer
  - `retention-explorer` — Retention Curve Explorer
  - `rv2021-shift` — RV2021 Composition Shift
  - `fiscal-waterfall` — Fiscal Incidence Waterfall

## Expected outputs

- `content/migrant-fiscal-impact.md` — updated with 5 `<div data-demo="..."></div>` tags inserted at the correct positions

## Execution instructions

### Step 1: Read the report markdown

Read `content/migrant-fiscal-impact.md` in its entirety. Understand the section structure and identify where each widget should be placed.

### Step 2: Identify placement positions

Look for either:
- Explicit widget placement comments/markers from Phase 2 (e.g., `<!-- WIDGET: npv-calculator -->` or `[WIDGET: npv-calculator]`)
- Natural positions based on content flow

The intended placements are:

1. **`npv-calculator`** — in the section discussing lifetime fiscal contributions / NPV analysis. This is likely in a section about "fiscal contribution by visa type" or "lifetime fiscal impact". Place it after the introductory paragraph of that section.

2. **`nationality-convergence`** — in the section discussing income convergence across nationalities. Likely near discussion of Table 8 data, nationality-specific income trends, or economic integration.

3. **`retention-explorer`** — in the section discussing migrant retention / departure rates. Likely near discussion of how many migrants stay vs leave, or cohort tracking.

4. **`rv2021-shift`** — in the section discussing the 2021 Residence Visa changes. Likely near discussion of RV2021, the one-off amnesty, or visa composition changes.

5. **`fiscal-waterfall`** — in the section discussing fiscal incidence methodology or the breakdown of taxes paid vs services received. Likely in a methodology or results section about fiscal decomposition.

### Step 3: Insert the tags

For each widget, insert the tag on its own line with blank lines before and after:

```markdown
...preceding paragraph text...

<div data-demo="widget-id"></div>

...following paragraph text...
```

Rules:
- Each tag MUST be on its own line
- There MUST be a blank line before and after the tag
- If there was a placement comment/marker, replace it with the tag (do not leave both)
- Do NOT place two widgets immediately adjacent to each other — there should be at least one paragraph of text between them

### Step 4: If the report markdown does not exist

If `content/migrant-fiscal-impact.md` does not exist, create a minimal placeholder:


## Handoff

With tags inserted, the MarkdownRenderer will attempt to load each widget when it encounters the tag. P4.3 completes the integration by updating page metadata. P5.3 will test that the widgets actually render.
