# Task P8.8: Validation gate

## Objective
Run a comprehensive validation of the synthetic population pipeline outputs against Phase 1 analytical results. This is the quality gate: if any metric exceeds its tolerance, the pipeline cannot proceed to Phase 9. The validation ensures the micro-simulation reproduces the macro-level results from the deterministic Phase 1 model, confirming that distributional detail has been added without introducing systematic bias.

## Inputs

### Phase 1 outputs (reference / ground truth)
- `data/output/npv-by-visa-age.json` — array of objects:
  - `visa` (str): visa type (e.g. "Skilled Work", "Family", "Australian")
  - `visa_code` (str): short code
  - `arrival_age` (int): 20, 25, 30, ..., 55
  - `npv` (float): lifecycle NPV in dollars
  - `nzborn_npv` (float): NZ-born equivalent NPV
  - `surplus` (float): npv - nzborn_npv
  - `trajectory` (array): year-by-year trajectory with `year`, `age`, `cumulative_npv`, `retention`, `nfi_year`

- `data/output/fiscal-components-by-migrant-type.json` — array of objects:
  - `year` (int), `age` (int), `retention` (float)
  - `direct_taxes`, `indirect_taxes`, `income_support`, `education`, `health`, `nfi` (float)
  - `type` (str): migrant type label

- `data/output/retention-curves-widget.json` — retention curve data

### Phase 2 synth-pop outputs (under test)
- `synth-pop/seed-population.parquet` — final synthetic population with all columns
- `data/output/synth-npv-distributions.json` — distributional NPV summaries
- `data/output/synth-population-summary.json` — aggregate population summary
- `data/output/synth-household-npv.json` — household widget data
- `data/output/synth-fiscal-distributions.json` — fiscal waterfall widget data

## Expected outputs
- `synth-pop/validation-report.md` — structured validation report with pass/fail for each metric
- If ALL metrics pass: report includes "GATE: PASS — proceed to Phase 9"
- If ANY metric fails: report includes "GATE: FAIL — see blockers below", and failures are also written to SCRATCHPAD under "## Blockers"

### Step 3: Generate validation report

```python
report_lines = [
    "# Synthetic population validation report",
    "",
    f"Date: {pd.Timestamp.now().strftime('%Y-%m-%d %H:%M')}",
    f"Population: {len(seed):,} individuals",
    f"Families: {seed['family_id'].nunique():,}",
    "",
    "## Metric results",
    "",
]

all_pass = True

# Add each metric result
for metric in metrics:
    status = "PASS" if metric['pass'] else "FAIL"
    if not metric['pass']:
        all_pass = False
    report_lines.append(f"### {metric['name']}")
    report_lines.append(f"**Status: {status}**")
    report_lines.append(f"Tolerance: {metric['tolerance']}")
    report_lines.append(f"Result: {metric['detail']}")
    report_lines.append("")

# Gate decision
report_lines.append("## Gate decision")
if all_pass:
    report_lines.append("**GATE: PASS** — proceed to Phase 9")
else:
    report_lines.append("**GATE: FAIL** — see blockers below")
    report_lines.append("")
    report_lines.append("### Blockers")
    for metric in metrics:
        if not metric['pass']:
            report_lines.append(f"- {metric['name']}: {metric['detail']}")

report = "\n".join(report_lines)

with open('synth-pop/validation-report.md', 'w') as f:
    f.write(report)

print(report)
```
