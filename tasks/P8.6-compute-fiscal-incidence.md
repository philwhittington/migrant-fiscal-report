# Task P8.6: Compute fiscal incidence

## Objective
Construct family units from the synthetic population, then compute individual fiscal incidence for every person. This applies the Wright and Nguyen (W&N) fiscal template with migrant-specific adjustments (healthy migrant effect on health spending, NZ Super residence rules, benefit stand-down, WFF restrictions for temporary visa holders). The output powers the household NPV widget and the fiscal waterfall widget.

## Inputs
- `synth-pop/seed-population.parquet` (P8.5 output) — columns: `id`, `age_start`, `visa_subcategory`, `visa_category`, `gross_income`, `income_tax`, `acc_levy`, `net_income`, `relationship`, `nationality`, `years_since_residence`
- `data/processed/wright-nguyen-fiscal-template.json` — W&N expenditure components by age band
- `data/processed/retention-curves-by-visa.json` — retention probabilities by visa type and year
- `synth-pop/utils.py` — import `get_wn_components`, `apply_migrant_adjustments`
- `synth-pop/config.py` — import all constants including `HEALTHY_MIGRANT_HEALTH_FACTOR`, `NZ_SUPER_RESIDENCE_YEARS`, `NZ_SUPER_AGE`, `BENEFIT_STANDOWN_YEARS`, `BENEFIT_STANDOWN_FACTOR`, `TEMP_VISA_CATEGORIES`
- `data/output/fiscal-components-by-migrant-type.json` — Phase 1 reference for validation (structure: array of objects with `year`, `age`, `retention`, `direct_taxes`, `indirect_taxes`, `income_support`, `education`, `health`, `nfi`, `type`)

## Expected outputs
1. `synth-pop/seed-population.parquet` — **updated in place** with new columns:
   - `family_id` (int): family unit identifier (primary applicant + linked spouse/children)
   - `health_cost` (float): age-specific health expenditure, adjusted by 0.85
   - `education_cost` (float): education expenditure (age-specific, mainly children/young adults)
   - `nz_super` (float): NZ Super entitlement (zero if <10 years residence or age <65)
   - `wff` (float): Working for Families (zero for temp visa holders)
   - `benefit` (float): income support / benefits (50% stand-down for first 2 years)
   - `indirect_tax` (float): GST and excise (scaled from income)
   - `other_expenditure` (float): remaining government expenditure (defence, infrastructure, etc.)
   - `total_revenue` (float): income_tax + acc_levy + indirect_tax
   - `total_expenditure` (float): health + education + nz_super + wff + benefit + other_expenditure
   - `net_fiscal_impact_annual` (float): total_revenue - total_expenditure

2. `data/output/synth-household-npv.json` — for the household widget. Array of representative household summaries:
   ```json
   [
     {
       "household_type": "Skilled couple + 2 children",
       "visa_category": "Resident",
       "arrival_age": 30,
       "members": [
         {"role": "primary", "age": 30, "income": 75000, "nfi": 8500},
         {"role": "spouse", "age": 28, "income": 35000, "nfi": -2000},

       "household_npv": 125000
     }
   ]
   ```

3. `data/output/synth-fiscal-distributions.json` — for the waterfall widget. Summary of fiscal components by visa category:
   ```json
   {
     "Resident": {
       "mean_income_tax": 12500,
   ```

## Execution instructions

### Step 1: Load all inputs
```python
import json
import numpy as np
import pandas as pd
from synth_pop.utils import get_wn_components, apply_migrant_adjustments
from synth_pop.config import *

seed = pd.read_parquet('synth-pop/seed-population.parquet')

with open('data/processed/wright-nguyen-fiscal-template.json') as f:
    wn_template = json.load(f)
```

### Step 2: Construct family units
Link primary applicants ("Self") to spouses and children with matching nationality and compatible ages.

```python
seed['family_id'] = -1
family_counter = 0

# Sort so Self comes first within each nationality group
seed = seed.sort_values(['nationality', 'age_start', 'relationship']).reset_index(drop=True)

# For each nationality group, match Self → Spouse → Children
for nat in seed['nationality'].unique():
    nat_mask = seed['nationality'] == nat

    selves = seed.loc[nat_mask & (seed['relationship'] == 'Self')].index.tolist()
    spouses = seed.loc[nat_mask & (seed['relationship'] == 'Presumed Spouse')].index.tolist()
    children = seed.loc[nat_mask & (seed['relationship'] == 'Presumed Child')].index.tolist()

    spouse_idx = 0
    child_idx = 0

    for self_id in selves:
        seed.loc[self_id, 'family_id'] = family_counter
        self_age = seed.loc[self_id, 'age_start']

        # Link spouse (within 10 years of age)
        if spouse_idx < len(spouses):
            sp = spouses[spouse_idx]
            sp_age = seed.loc[sp, 'age_start']
            if abs(sp_age - self_age) <= 10:
                seed.loc[sp, 'family_id'] = family_counter
                spouse_idx += 1

        # Link up to 3 children (age < self - 15)
        linked_children = 0
        while child_idx < len(children) and linked_children < 3:
            ch = children[child_idx]
            ch_age = seed.loc[ch, 'age_start']
            if ch_age < self_age - 15:
                seed.loc[ch, 'family_id'] = family_counter
                child_idx += 1
                linked_children += 1
            else:
                break

        family_counter += 1

    # Assign remaining unlinked spouses/children to their own families
    unlinked = seed.loc[nat_mask & (seed['family_id'] == -1)].index
    for idx in unlinked:
        seed.loc[idx, 'family_id'] = family_counter
        family_counter += 1
```

### Step 3: Compute fiscal components for each individual
```python
for col in ['health_cost', 'education_cost', 'nz_super', 'wff', 'benefit',
            'indirect_tax', 'other_expenditure']:
    seed[col] = 0.0

for idx, row in seed.iterrows():
    # Get base W&N components for this age
    wn = get_wn_components(row['age_start'], wn_template)

    # Apply migrant adjustments
    adjusted = apply_migrant_adjustments(
        wn, row['visa_category'], row['years_since_residence'], row['age_start']
    )

    seed.at[idx, 'health_cost'] = adjusted.get('health', 0)
    seed.at[idx, 'education_cost'] = adjusted.get('education', 0)
    seed.at[idx, 'nz_super'] = adjusted.get('nz_super', 0)
    seed.at[idx, 'wff'] = adjusted.get('wff', 0)
    seed.at[idx, 'benefit'] = adjusted.get('benefit', 0)
    seed.at[idx, 'other_expenditure'] = adjusted.get('other', 0)

    # Indirect tax: scale from gross income (approx 15% of consumption)
    seed.at[idx, 'indirect_tax'] = row['gross_income'] * 0.15
```

**Performance note**: The row-wise loop above is very slow on 500k rows. Vectorise using groupby on `(age_start, visa_category, years_since_residence)` to compute components per group and broadcast, rather than per individual. Only income-dependent components (indirect_tax) need individual-level calculation.

### Step 4: Compute totals
```python
seed['total_revenue'] = seed['income_tax'] + seed['acc_levy'] + seed['indirect_tax']
seed['total_expenditure'] = (seed['health_cost'] + seed['education_cost'] +
                             seed['nz_super'] + seed['wff'] + seed['benefit'] +
                             seed['other_expenditure'])
seed['net_fiscal_impact_annual'] = seed['total_revenue'] - seed['total_expenditure']
```
