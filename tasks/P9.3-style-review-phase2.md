# Task P9.3: Style review of Phase 2 content updates

## Objective

Review all updated report content for adherence to HW report style, appropriate hedging of synthetic estimates, tonal consistency between Phase 1 and Phase 2 prose, and factual accuracy of sourced numbers. This is the quality gate before widgets are wired in.

## Inputs

- `content/migrant-fiscal-impact.md` — the full report after P9.1 and P9.2 edits
- `data/output/synth-npv-distributions.json` — source data for distributional claims
- `data/output/synth-fiscal-distributions.json` — source data for fiscal component distributions
- `data/output/synth-household-npv.json` — source data for household claims
- `data/output/synth-population-summary.json` — aggregate statistics
- `synth-pop/validation-report.md` — validation statistics cited in Section 2.6

## Expected outputs

- `content/migrant-fiscal-impact.md` — corrected in-place (fixes applied directly)
- No separate report file is produced; findings are applied as edits

## Execution instructions

### Step 1: Read the full report

Read `content/migrant-fiscal-impact.md` in its entirety. Take note of the overall structure, heading hierarchy, and tone of the original Phase 1 content.

### Step 2: Check heading style

Verify all headings use sentence case (first word capitalised, rest lowercase unless proper noun). Fix any headings that use title case or all-caps. This includes the new subsection 2.6 and any new sub-headings added in P9.2.

### Step 3: Check for vague qualifiers

Search the text for vague qualifiers that weaken analytical prose. Flag and fix instances of:
- "significant" (replace with specific magnitude)
- "substantial" (replace with specific quantity)
- "quite", "rather", "fairly" (remove or replace with specifics)
- "a lot of", "many" (replace with specific numbers)
- "it is clear that", "obviously" (remove — let the evidence speak)

This applies to both Phase 1 and Phase 2 content for consistency.

### Step 4: Verify all numbers are sourced

For every numerical claim in Phase 2 content (added in P9.1 and P9.2):
1. Read the corresponding JSON source file
2. Confirm the number matches
3. If a number does not match any source, flag it and either correct it or remove it

For Phase 1 content, spot-check 5-10 key numbers against their source files in `data/output/`.

### Step 5: Check hedging language for synthetic estimates

Every Phase 2 distributional claim must use appropriate hedging. Scan for and fix:

**Must not appear in Phase 2 claims:**
- "migrants contribute" → change to "our synthetic population estimates suggest migrants contribute"
- "the data shows" → change to "the model indicates"
- "we observe" → change to "the estimated distribution shows"
- "on average" without context → specify whether this is the Phase 1 group mean or Phase 2 synthetic mean

**Must appear at least once:**
- A clear statement that Phase 2 results are model-generated synthetic estimates, not observed individual data
- A reference to Section 2.6 for methodology details

### Step 6: Check tonal consistency

Read Sections 3-7 sequentially and verify:
- Phase 1 prose (group means) and Phase 2 prose (distributions) are clearly distinguished
- The transition from group-mean findings to distributional findings is smooth
- Phase 2 paragraphs feel additive and enriching, not contradictory
- The overall confidence level is consistent — Phase 2 claims should not be stated with more certainty than Phase 1 claims

### Step 7: Check cross-references and flow

Verify:
- Section 2.6 is referenced from the distributional paragraphs in later sections
- No orphaned references to sections, figures, or data that do not exist
- The conclusion (Section 9) acknowledges Phase 2 distributional findings if it discusses overall findings

### Step 8: Apply all fixes

Make all corrections directly in `content/migrant-fiscal-impact.md`. Do not create a separate corrections file.

## Self-check

1. All headings use sentence case
2. No vague qualifiers remain without specific quantities
3. Every Phase 2 number traces to a JSON source file
4. All synthetic estimate claims use hedged language
5. Phase 1 and Phase 2 prose are tonally consistent
6. Cross-references are intact
7. The document reads as a cohesive whole, not a Frankenstein of two phases
