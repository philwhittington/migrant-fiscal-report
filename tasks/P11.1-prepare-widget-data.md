# Task P11.1: Prepare and verify widget data files

## Objective

Verify all synthetic population JSON data files exist in `data/output/`, are correctly formatted, and copy them to the client public directory for serving. Ensure no file exceeds the 1MB size limit for performant browser loading.

## Inputs

- `data/output/synth-npv-distributions.json` — NPV distributions by visa type and arrival age
- `data/output/synth-fiscal-distributions.json` — fiscal component distributions
- `data/output/synth-household-npv.json` — household-level NPV data
- `data/output/synth-population-summary.json` — aggregate summary statistics
- `script/generate-static-data.ts` — the data pipeline script that copies output files to `client/public/data/`
- `client/public/data/` — the directory served at `/data/` in the browser

## Expected outputs

- All 4 synth-*.json files present in `client/public/data/`
- All existing Phase 1 data files still present in `client/public/data/`
- Size report confirming no file exceeds 1MB

## Execution instructions

### Step 1: Verify source files exist

```bash
cd /Users/PhilWhittington/Projects/migrant-fiscal-report
ls -la data/output/synth-*.json
```

Confirm all 4 files exist:
- `synth-npv-distributions.json`
- `synth-fiscal-distributions.json`
- `synth-household-npv.json`
- `synth-population-summary.json`

If any file is missing, stop and report. These should have been created by Phase 8 tasks.

### Step 2: Validate JSON structure

For each file, read it and verify:
1. It is valid JSON (no syntax errors)
2. It contains the expected top-level keys
3. Data is populated (not empty arrays/objects)
4. Numbers are reasonable (no NaN, no Infinity, no absurdly large values)

Run a quick validation:
```bash
for f in data/output/synth-*.json; do
  echo "--- $f ---"
  python3 -c "import json; d=json.load(open('$f')); print(f'Keys: {list(d.keys())}')" 
done
```

### Step 3: Check file sizes

```bash
ls -lh data/output/synth-*.json
```

If any file exceeds 1MB, reduce it:
- For histogram data (`synth-npv-distributions.json`): reduce bin count from 50 to 30 bins
- For per-visa-type data: ensure no redundant precision (round to nearest dollar, not cent)
- For large arrays: consider whether all arrival ages are needed or if a subset suffices

Apply trimming with a Python one-liner if needed:
```bash
python3 -c "
import json
d = json.load(open('data/output/synth-npv-distributions.json'))
# Trim logic here
json.dump(d, open('data/output/synth-npv-distributions.json', 'w'), separators=(',', ':'))
"
```

### Step 4: Run the data pipeline

```bash
cd /Users/PhilWhittington/Projects/migrant-fiscal-report
npx tsx script/generate-static-data.ts
```

If this script does not handle synth-* files, manually copy them:
```bash
cp data/output/synth-*.json client/public/data/
```

### Step 5: Verify destination files

```bash
ls -lh client/public/data/
```

Confirm:
- All 4 synth-*.json files are present in `client/public/data/`
- All existing Phase 1 data files are still present (not overwritten or deleted)
- File sizes match the source files

### Step 6: List all data files with sizes

```bash
echo "=== All data files ==="
ls -lh client/public/data/*.json | awk '{print $5, $NF}'
echo "=== Total ==="
du -sh client/public/data/
```

Document the file list and sizes for the QA report.
