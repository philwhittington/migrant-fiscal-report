# Task P5.4: Final build and review

## Objective

Run the final production build, verify it succeeds, and produce a learnings document summarising what worked, what was difficult, and what assumptions need human review.

## Inputs

- The entire project at `/Users/PhilWhittington/Projects/migrant-fiscal-report/`
- QA reports from P5.1, P5.2, P5.3 (in `.agent/`)

## Expected outputs

- A successful production build in `dist/`
- `.agent/LEARNINGS.md` — summary of the build process, quality issues, and assumptions
- `.agent/SCRATCHPAD.md` — updated with final state

## Execution instructions

### Step 1: Clean build

```bash
cd /Users/PhilWhittington/Projects/migrant-fiscal-report
rm -rf dist/
npm run build
```

If the build fails:
1. Read the error output in full.
2. Fix the root cause (do not suppress errors).
3. Re-run until the build succeeds.
4. Document every fix in the learnings file.

### Step 2: Verify build output

```bash
ls -la dist/
ls -la dist/public/ 2>/dev/null || ls -la dist/
```

Check that:
- An `index.html` exists in the output
- JavaScript bundles exist (`.js` files)
- CSS files exist
- The `data/` directory exists in the build output with all JSON files

If the build output structure is unclear, check `vite.config.ts` for the output directory configuration.

### Step 3: Check build size

```bash
du -sh dist/
find dist/ -name "*.js" -exec du -sh {} \;
find dist/ -name "*.json" -exec du -sh {} \;
```

Flag any concerns:
- Total build > 10MB: investigate which files are large
- Any single JS bundle > 500KB: may need code splitting
- Any JSON data file > 1MB: may need optimisation

### Step 4: Read QA reports

Read the three QA reports if they exist:
- `.agent/qa-data-report.md`
- `.agent/qa-prose-report.md`
- `.agent/qa-widget-report.md`

Compile a summary of:
- Total issues found across all QA stages
- Issues resolved
- Issues still outstanding
- Items flagged for human review
