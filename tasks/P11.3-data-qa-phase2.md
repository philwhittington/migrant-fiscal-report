# Task P11.3: Data QA for Phase 2 distributional claims

## Objective

Cross-check every distributional claim in the report text against the synthetic population output files. Verify percentiles, probability statements, and histogram data are correctly cited. Apply sanity bounds: no Phase 2 number should exceed 2x or fall below 0.5x the corresponding Phase 1 equivalent. Write a QA report documenting all checks.

## Inputs

- `content/migrant-fiscal-impact.md` — the full report with Phase 2 distributional text
- `data/output/synth-npv-distributions.json` — NPV distribution data
- `data/output/synth-fiscal-distributions.json` — fiscal component distributions
- `data/output/synth-household-npv.json` — household NPV data
- `data/output/synth-population-summary.json` — aggregate summary
- `data/output/npv-by-visa-age.json` — Phase 1 NPV data (for sanity comparison)
- `data/output/fiscal-components-by-migrant-type.json` — Phase 1 fiscal data (for sanity comparison)

## Expected outputs

- `.agent/qa-data-report-phase2.md` — QA report documenting all checks, findings, and pass/fail status

## Execution instructions

### Step 1: Extract all numerical claims from the report

Read `content/migrant-fiscal-impact.md`. For every numerical claim added in Phase 2 (P9.1 and P9.2), extract:
- The number
- The context/sentence
- The section it appears in
- The expected source file

Build a checklist of claims to verify. Expected claim types:
- Percentile values (p10, p25, p50, p75, p90)
- Probability statements ("X% of Y are Z")
- Distributional range statements ("p10-p90 range of $X to $Y")
- Household NPV values
- Validation statistics (MAPE, correlation)
- Inequality metrics (Gini, IQR)

### Step 2: Verify each claim against source data

For each numerical claim:
1. Read the relevant JSON source file
2. Navigate to the specific data point
3. Compare the report number to the source number
4. Record: PASS (exact match or within rounding), FAIL (mismatch), or MISSING (source data not found)

### Step 3: Apply sanity bounds against Phase 1

For each visa type where both Phase 1 and Phase 2 values exist:
1. Read the Phase 1 mean NPV from `data/output/npv-by-visa-age.json`
2. Read the Phase 2 mean NPV from `data/output/synth-npv-distributions.json`
3. Compute the ratio: Phase2_mean / Phase1_mean
4. Flag if the ratio exceeds 2.0 or falls below 0.5

Do the same for fiscal components:
1. Read Phase 1 component values from `data/output/fiscal-components-by-migrant-type.json`
2. Read Phase 2 mean component values from `data/output/synth-fiscal-distributions.json`
3. Flag any component where the ratio exceeds 2.0 or falls below 0.5

### Step 4: Verify internal consistency

Check that:
- Percentiles are monotonically increasing (p10 < p25 < p50 < p75 < p90) for every distribution
- Probability of net contribution is consistent with the distribution (e.g., if median NPV > 0, then pctNetContributor should be > 50%)
- Household NPV sums match component sums (primary + spouse + children = total)
- Population shares in summary file sum to approximately 100%

### Step 5: Check validation statistics

Read `synth-pop/validation-report.md` and verify the MAPE and correlation statistics cited in Section 2.6 match the validation report.
